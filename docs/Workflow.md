### Objective

<br>

**Checklist**

<br>
  
```json
1. Short report about progress
 
2. If neccessary assign new tasks
 
3. Create new tasks as new issues in the boar

    3.1 Decompose tasks into small sub-issues such that every issue can be finished in one to two days

    3.2 Create new branch for every issue

    3.3 Write tests for functions and/or properties in reference to the given issue

4. Use one or more testing styles of your choice like...

    4.1 unit-tests            (two tests per function)

    4.2 property-based-tests  (test functions with properties - every function occurs at least once)

5. Merge code after every finished, reviewed as well as tested issue *

    5.1 Create pull-request

    5.2 Other developer should review code and then merge the pull request

```
